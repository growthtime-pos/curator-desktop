# Release Note Draft

Use this skill to turn technical changes into short release-note copy.

## Workflow

1. Identify the end-user facing changes.
2. Group changes by feature area and omit low-signal implementation noise.
3. Prefer short bullets and plain language.

Read [references/template.md](references/template.md) for the preferred output structure.
